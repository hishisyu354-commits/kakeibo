# カード自動同期（ほぼリアルタイム）セットアップ手順

食費専用カードの「利用通知メール」を、家計簿アプリに数分で自動反映させるための設定です。
一度だけの設定で、以後は開くたび＆約1分ごとに自動で取り込まれます（手入力なし）。

前提：**食費専用の三菱UFJ-VISAデビットカード**（このデビットの決済＝すべて食費、という運用）。

---

## 0. デビットカードの準備
1. 三菱UFJ銀行の口座に付く **三菱UFJ-VISAデビットカード** を用意し、**食費専用**にする
2. 三菱UFJダイレクト（VISAデビット会員用Web）で **通知先Eメールアドレス＝普段使う Gmail** に登録。`@debit.bk.mufg.jp` を受信できるようにしておく
3. そのデビットで一度食費を払い、差出人 `mail@debit.bk.mufg.jp`／件名「【三菱UFJ-VISAデビット】ご利用のお知らせ」のメールが Gmail に届くのを確認

## 1. Apps Script を作る
1. https://script.google.com を開く → 「新しいプロジェクト」
2. 既定の `Code.gs` の中身を全部消し、この folder の **Code.gs** の中身を貼り付け
3. 冒頭の `CONFIG` を編集：
   - `TOKEN` … 好きな長いランダム文字列（例 `kakeibo-8f3a9c2b7d`）。人に教えない
   - `CARD_LAST4` … 食費カードの**下4桁**（例 `1234`）。分からなければ空 `''` のままでも可
4. 保存（💾）

## 2. 動作確認（メール読み取りの権限承認）
1. 関数の選択で `testFetch` を選び「実行」
2. 初回は Google の権限承認ダイアログ → 自分のアカウントで許可（Gmail 読み取り権限）
3. 「実行ログ」に `取得件数: N` とサンプルが出れば成功
   - 0件や金額がおかしい場合 → 実物メールの本文を菱沼さんが Claude に貼れば、`parseMail` の正規表現を実物に合わせて直します

## 3. Web アプリとしてデプロイ
1. 右上「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
2. 「次のユーザーとして実行」= **自分**、「アクセスできるユーザー」= **全員**
   （アプリが未ログインで読めるようにするため。保護は URL 内の `token` で行います）
3. デプロイ → 表示される **ウェブアプリ URL**（`https://script.google.com/macros/s/.../exec`）をコピー

## 4. アプリに URL を貼る
1. 家計簿アプリ → **設定 → 自動同期（カード連携）**
2. 「同期URL」に、コピーした URL の末尾に **`?token=あなたのTOKEN`** を付けて貼る：
   `https://script.google.com/macros/s/XXXX/exec?token=kakeibo-8f3a9c2b7d`
3. 「保存」→「今すぐ同期」→ 「◯件を同期しました」と出れば完了 🎉
   以後は開くたび＆約1分ごとに自動で増えます。

---

## 仕様・注意
- **食費専用カード前提**なので、届いた通知は全部そのまま食費として追加されます
- **重複しません**（メールのIDで管理。何度同期しても二重計上なし）
- **起算日より前・未来の日付は取り込みません**
- **携帯/公共料金の継続払い・ETC・一部電子マネーは通知対象外**（＝食費だけ通る運用に好都合）
- 店名は通知に載っていれば入りますが、無ければ「カード」と表示（食費判定には不要）
- URL＋token を知る人は明細（金額）を読めます。URLは人に共有しないでください
- リアルタイムではなく**数分の遅れ**があります（メール到着＋ポーリング間隔）

## うまくいかない時
- 「今すぐ同期」で失敗 → URL と `?token=` が正しいか、デプロイのアクセス設定が「全員」か確認
- 0件 → `CARD_LAST4` を空にして再確認 / `testFetch` のログを確認 / 実物メール本文を Claude に貼って解析調整
