# Apps Script セットアップ手順（家計簿の自動同期 ＋ 食材ストッカーのAI連携）

1つの Google Apps Script プロジェクトで、次の両方が動きます。**設定は一度だけ**です。

- 💰 **食費家計簿**：食費専用カードの利用通知メールを、開くたび＆約1分ごとに自動取り込み
- 🧺 **食材ストッカー**：写真から賞味期限・食材名を読み取り／在庫から料理を提案／期限が近いと毎朝メール通知
  （AIは Google **Gemini API の無料枠**を使用。クレジットカード登録は不要です）

---

## 1. Apps Script プロジェクトを作る

1. https://script.google.com を開く → 「新しいプロジェクト」
2. 既定の `Code.gs` の中身を全部消し、この folder の **Code.gs** の中身を貼り付け
3. 冒頭の `CONFIG` を編集：
   - `TOKEN` … 好きな長いランダム文字列（例 `kakeibo-XXXXXXXXXXXX` の X を自分で置き換える）。**この例をそのまま使わない**・人に教えない
   - `GEMINI_API_KEY` … 手順2で発行するキー（食材ストッカーを使う場合）
   - `CARD_LAST4` … 食費カードの**下4桁**（例 `1234`）。分からなければ空 `''` のままでも可
4. 左メニューの **プロジェクトの設定（⚙）** → タイムゾーンが **「(GMT+09:00) 東京」** になっているか確認（通知メールの時刻に影響）
5. 保存（💾）

## 2. Gemini APIキーを無料発行（食材ストッカー用）

1. https://aistudio.google.com/apikey を開く（自分のGoogleアカウントでOK・**無料**・カード不要）
2. 「APIキーを作成」→ 表示されたキー（`AIza...`）をコピー
3. Code.gs の `GEMINI_API_KEY` に貼り付けて保存
4. 動作確認：関数の選択で `testGemini` を選び「実行」→ 初回は権限承認 → ログに `{"ok":true}` が出れば成功

※ 無料枠の目安は1日数百リクエスト。写真読み取り数回＋料理提案なら余裕です。

## 3. 動作確認（メール読み取りの権限承認・家計簿用）

1. 関数の選択で `testFetch` を選び「実行」
2. 初回は Google の権限承認ダイアログ → 自分のアカウントで許可（Gmail 読み取り権限）
3. 「実行ログ」に `取得件数: N` とサンプルが出れば成功
   - 0件や金額がおかしい場合 → 実物メールの本文を菱沼さんが Claude に貼れば、`parseMail` の正規表現を実物に合わせて直します

## 4. 期限通知メールを有効化（食材ストッカー用・任意）

1. 関数の選択で `setupNotifyTrigger` を選び「実行」（1回だけ）
2. 以後、**毎朝8時台**に在庫をチェックし、期限切れ・期限が近い食材が**ある日だけ**メールが届きます
   （宛先は自分のGmail。iPhoneのGmail通知でそのまま気づけます）

## 5. Web アプリとしてデプロイ

1. 右上「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
2. 「次のユーザーとして実行」= **自分**、「アクセスできるユーザー」= **全員**
   （アプリが未ログインで読めるようにするため。保護は URL 内の `token` で行います）
3. デプロイ → 表示される **ウェブアプリ URL**（`https://script.google.com/macros/s/.../exec`）をコピー

※ Code.gs をあとから書き換えたら「デプロイ」→「デプロイを管理」→ ✏️ → バージョン「新しいバージョン」→ デプロイ、で反映されます（URLは変わりません）。

## 6. アプリに URL を貼る（両アプリ共通・同じURL）

URL の末尾に **`?token=あなたのTOKEN`** を付けた形を使います：
`https://script.google.com/macros/s/XXXX/exec?token=あなたのTOKEN`

- 💰 家計簿 → **設定 → 自動同期（カード連携）** の「同期URL」に貼る → 保存 → 「今すぐ同期」
- 🧺 食材ストッカー → **設定 → AI連携** の「連携URL」に貼る → 保存 → 「接続テスト」
  「✅ 接続OK（AI読み取り・提案も利用可能）」と出れば完了 🎉

## 仕様・注意

- **食費専用カード前提**なので、届いた通知は全部そのまま食費として追加されます
- **重複しません**（メールのIDで管理。手入力・CSV取り込み分とも日付+金額で照合して二重計上を防ぎます）
- **起算日より前・未来の日付は取り込みません**
- **携帯/公共料金の継続払い・ETC・一部電子マネーは通知対象外**（＝食費だけ通る運用に好都合）
- 店名は通知に載っていれば入りますが、無ければ「カード」と表示（食費判定には不要）
- URL＋token を知る人は明細（金額）や在庫リストを読めます。URLは人に共有しないでください
- 家計簿の同期はリアルタイムではなく**数分の遅れ**があります（メール到着＋ポーリング間隔）
- 食材の写真は読み取りのために Google（Gemini API）へ送信されます。無料枠の写真は品質向上に使われることがあるため、**食品以外が写り込まない**ようにすると安心です

## うまくいかない時

- 「今すぐ同期」「接続テスト」で失敗 → URL と `?token=` が正しいか、デプロイのアクセス設定が「全員」か確認
- 家計簿が0件 → `CARD_LAST4` を空にして再確認 / `testFetch` のログを確認 / 実物メール本文を Claude に貼って解析調整
- 写真読み取りが失敗 → `testGemini` を実行してログを確認。`429` は無料枠の一時上限（時間をおく）、モデル名エラーは `GEMINI_MODEL` を `'gemini-2.5-flash-lite'` に変更
- 通知メールが来ない → `setupNotifyTrigger` を実行済みか確認（該当食材が無い日はメール自体が届きません）
- 削除したはずの食材の通知が届く → アプリの「設定 → 全データを削除」はサーバー側も消しますが、通信に失敗した場合は残ることがあります。Apps Script エディタで `PropertiesService.getScriptProperties().deleteProperty('PANTRY_DATA')` を1回実行すれば止まります
